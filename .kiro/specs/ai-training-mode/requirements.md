# Requirements Document

## Introduction

This feature adds an AI-powered training mode to Train with Joe. Unlike existing training modes (text input and multiple choice) which test direct word-to-translation recall, the AI Training Mode uses Amazon Bedrock to generate contextual language exercises based on selected vocabulary words. Exercises include fill-in-the-blank sentences, verb conjugation, preposition selection, and other grammar-focused challenges. This gives users a deeper way to practice vocabulary in realistic language contexts rather than isolated word pairs.

## Glossary

- **AI_Training_Mode**: A new training mode where exercises are generated by the AI_Service based on selected vocabulary words, testing contextual language usage rather than direct translation recall
- **AI_Exercise**: A single AI-generated question with a prompt sentence, a set of answer options, and one correct answer
- **Exercise_Type**: The category of an AI-generated exercise (e.g., verb conjugation, preposition selection, fill-in-the-blank, sentence completion)
- **AI_Service**: The existing Amazon Bedrock integration service responsible for generating AI content, extended to generate contextual exercises
- **Training_Service**: The existing backend service responsible for creating, starting, and managing trainings
- **Training**: A reusable vocabulary practice session with a name, mode, direction, and associated vocabulary lists
- **Training_Execution**: A single run of a Training, tracking answers, scores, and timing
- **Vocabulary_List**: A user-owned collection of vocabulary words with translations, definitions, and example sentences
- **Vocabulary_Word**: A single word entry in a Vocabulary_List, containing the word, translation, definition, part of speech, and example sentence
- **Frontend_App**: The Flutter mobile application used by the end user
- **GraphQL_API**: The AWS AppSync-based API for client-server communication

## Requirements

### Requirement 1: AI Training Mode Definition

**User Story:** As a user, I want a new AI-powered training mode, so that I can practice vocabulary in realistic language contexts beyond simple word-translation recall.

#### Acceptance Criteria

1. THE GraphQL_API SHALL include an AI_TRAINING value in the TrainingMode enum
2. THE Training domain model SHALL support AI_TRAINING as a valid TrainingMode value
3. WHEN a user creates a Training with mode set to AI_TRAINING, THE Training_Service SHALL store the Training with the AI_TRAINING mode

### Requirement 2: AI Exercise Generation

**User Story:** As a user, I want the system to generate contextual exercises from my vocabulary words, so that I can practice grammar, prepositions, and verb forms in sentence context.

#### Acceptance Criteria

1. WHEN a user starts a Training with AI_TRAINING mode, THE AI_Service SHALL generate one AI_Exercise per selected Vocabulary_Word
2. WHEN generating an AI_Exercise, THE AI_Service SHALL use the Vocabulary_Word fields (word, translation, definition, part of speech, example sentence) as context for exercise creation
3. WHEN generating an AI_Exercise, THE AI_Service SHALL produce a prompt sentence with a blank or selection point, a list of answer options (minimum 3, maximum 5), and the index of the correct option
4. WHEN generating AI_Exercises, THE AI_Service SHALL vary the Exercise_Type across the set of exercises (e.g., verb conjugation, preposition selection, fill-in-the-blank, sentence completion)
5. WHEN generating AI_Exercises, THE AI_Service SHALL use the source and target languages from the associated Vocabulary_List to produce exercises in the appropriate language
6. WHEN generating an AI_Exercise, THE AI_Service SHALL include the Exercise_Type label in the exercise data so the Frontend_App can display it to the user
7. IF the AI_Service fails to generate exercises, THEN THE Training_Service SHALL return a descriptive error message and not create a Training_Execution

### Requirement 3: AI Exercise Data Model

**User Story:** As a developer, I want a structured data model for AI-generated exercises, so that the frontend can render exercises consistently and the backend can validate answers.

#### Acceptance Criteria

1. THE GraphQL_API SHALL define an AIExercise type containing a prompt string, an options list of strings, a correctOptionIndex integer, an exerciseType string, and a sourceWord string
2. THE TrainingExecution type in the GraphQL_API SHALL include an optional aiExercises field containing a list of AIExercise items
3. THE Training domain model SHALL include an AIExercise interface with prompt, options, correctOptionIndex, exerciseType, and sourceWord fields
4. THE TrainingExecution domain model SHALL include an optional aiExercises field

### Requirement 4: AI Training Execution Flow

**User Story:** As a user, I want to start an AI training and answer generated exercises, so that I can practice and get scored on contextual language usage.

#### Acceptance Criteria

1. WHEN a user starts a Training with AI_TRAINING mode, THE Training_Service SHALL select words from the associated Vocabulary_Lists (respecting word count and unit filters)
2. WHEN a user starts a Training with AI_TRAINING mode, THE Training_Service SHALL call the AI_Service to generate exercises for the selected words
3. WHEN a user starts a Training with AI_TRAINING mode, THE Training_Service SHALL store the generated AI_Exercises on the Training_Execution
4. WHEN a user submits an answer for an AI_TRAINING execution, THE Training_Service SHALL compare the submitted answer against the correctOptionIndex of the corresponding AI_Exercise
5. WHEN a user completes all AI_Exercises in a Training_Execution, THE Training_Service SHALL mark the execution as completed with correct and incorrect counts
6. WHEN a user starts a Training with AI_TRAINING mode and fewer than 1 word is available, THE Training_Service SHALL return a descriptive error message and not create a Training_Execution

### Requirement 5: AI Training with Randomized Mode

**User Story:** As a user, I want to combine AI training mode with randomized word selection, so that I get fresh AI-generated exercises from different words each session.

#### Acceptance Criteria

1. WHEN a user starts a Randomized Training with AI_TRAINING mode, THE Training_Service SHALL dynamically select words from the attached Vocabulary_Lists before generating AI_Exercises
2. WHEN a user starts a Randomized Training with AI_TRAINING mode, THE Training_Service SHALL generate new AI_Exercises for the newly selected words each time
3. THE Training_Service SHALL support AI_TRAINING mode for both static and randomized trainings using the same exercise generation logic

### Requirement 6: AI Exercise Generation Rate Limiting

**User Story:** As a system operator, I want AI exercise generation to respect rate limits, so that the Bedrock service is not overwhelmed and costs remain controlled.

#### Acceptance Criteria

1. WHEN generating AI_Exercises, THE AI_Service SHALL check the existing per-user rate limit (10 requests per minute) before calling Bedrock
2. IF the user has exceeded the rate limit, THEN THE AI_Service SHALL return a descriptive rate limit error message
3. WHEN generating AI_Exercises, THE AI_Service SHALL log the usage (user ID, operation name, estimated token count) for monitoring

### Requirement 7: AI Exercise Response Parsing

**User Story:** As a developer, I want robust parsing of AI-generated exercise responses, so that malformed AI output does not break the training flow.

#### Acceptance Criteria

1. WHEN the AI_Service receives a response from Bedrock, THE AI_Service SHALL parse the response as a JSON array of AI_Exercise objects
2. IF the Bedrock response contains invalid JSON, THEN THE AI_Service SHALL return a descriptive parsing error
3. IF a parsed AI_Exercise is missing required fields (prompt, options, correctOptionIndex), THEN THE AI_Service SHALL exclude that exercise and log a warning
4. IF all parsed AI_Exercises are invalid, THEN THE AI_Service SHALL return a descriptive error message
5. FOR ALL valid AI_Exercise objects, parsing the Bedrock response then serializing then parsing again SHALL produce equivalent AI_Exercise objects (round-trip property)

### Requirement 8: Frontend AI Training Creation

**User Story:** As a user, I want to select AI Training Mode when creating a training, so that I can set up AI-powered practice sessions.

#### Acceptance Criteria

1. WHEN a user is creating a training, THE Frontend_App SHALL display AI Training Mode as a selectable training mode option alongside Text Input and Multiple Choice
2. WHEN the user selects AI Training Mode, THE Frontend_App SHALL send the mode as AI_TRAINING in the createTraining mutation
3. WHEN the user selects AI Training Mode, THE Frontend_App SHALL allow the user to select vocabulary lists, set word count, and optionally enable randomized mode (same as other modes)

### Requirement 9: Frontend AI Exercise Display

**User Story:** As a user, I want to see AI-generated exercises presented clearly during training, so that I can understand and answer each question.

#### Acceptance Criteria

1. WHEN displaying an AI_Exercise, THE Frontend_App SHALL show the exercise type label, the prompt sentence, and the answer options as selectable buttons
2. WHEN the user selects an answer option, THE Frontend_App SHALL submit the selected option index as the answer
3. WHEN the user submits an answer, THE Frontend_App SHALL display whether the answer was correct or incorrect along with the correct answer
4. WHEN all exercises are completed, THE Frontend_App SHALL display the training results summary with correct and incorrect counts

### Requirement 10: Backward Compatibility

**User Story:** As a user, I want my existing TEXT_INPUT and MULTIPLE_CHOICE trainings to continue working unchanged, so that the new AI mode does not affect my current workflow.

#### Acceptance Criteria

1. WHEN a user starts a Training with TEXT_INPUT or MULTIPLE_CHOICE mode, THE Training_Service SHALL follow the existing execution flow without invoking the AI_Service
2. THE Training_Service SHALL treat all existing Training entities that lack the AI_TRAINING mode as TEXT_INPUT or MULTIPLE_CHOICE trainings (existing behavior preserved)
3. THE GraphQL_API SHALL remain backward compatible by adding the new AI_TRAINING enum value and AIExercise type without modifying existing types or fields
